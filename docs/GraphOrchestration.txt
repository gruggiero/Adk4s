AI generated guide
Motivation
ADK4S needs to handle conditional routing in orchestration graphs - deciding which node to execute next based on runtime data. Without branches, graphs would be limited to linear flows. The system provides two levels: low-level Branch for graph routing [3b] and high-level WIOBranch for Workflows4s integration [3d].

Details
Graph-level branching starts with Graph.addBranch() [3a] which attaches a Branch to a node. The most common is Branch.binary() [3b] which creates an InvokeBranch that evaluates a predicate and routes to either ifTrue or ifFalse nodes [3c].

WIO-level branching provides Workflows4s integration through WIOBranch.fork() [3d]. This creates a WIO workflow using WIO.build().fork() [3e] with matchCondition() routing that executes either the onTrue or onFalse workflow based on the condition.

The key difference is that Branch operates at the graph compilation level (routing between graph nodes), while WIOBranch operates at the workflow execution level (routing within WIO workflows). Both solve the same conditional routing problem but at different abstraction layers.
